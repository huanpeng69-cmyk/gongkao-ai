## 问题1: 编译错误 - 中文引号语法问题

### 原始问题
```
app/api/ai/route.ts(14,20): error TS1005: ',' expected.
```

### 根本原因
TypeScript 不允许在双引号字符串中使用转义的双引号来嵌入中文引号。原代码：
```typescript
standard: "题干含"削弱""反驳"..."  // ❌ 错误
```

### 解决方案
使用单引号包裹包含双引号的字符串：
```typescript
standard: '题干含"削弱""反驳""质疑""最能削弱"等关键词'  // ✅ 正确
```

**文件**: `app/quiz/page.tsx`
```typescript
useEffect(() => {
  if (!selectedItem) return;
  const imageSources: string[] = [];
  // 收集当前题目的所有图片
  if (selectedItem.type === "single") {
    imageSources.push(...getQuestionImageSources(selectedItem.question));
  } else {
    selectedItem.group.questions.forEach((q) => {
      imageSources.push(...getQuestionImageSources(q));
    });
  }
  // 预加载
  if (imageSources.length > 0) {
    preloadImages(imageSources).catch(() => {});
  }
}, [selectedItem]);
```

## 技术细节

### 修复要点

1. **中文引号处理**
   - 问题：TypeScript 不支持在双引号字符串中使用 `\"` 转义双引号
   - 解决：改用单引号包裹包含双引号的字符串
   - 示例：`'题干含"削弱""反驳"等关键词'`

2. **对象键的中文支持**
   - JavaScript/TypeScript 支持无引号的中文标识符
   - 语法：`{ 削弱论证: { ... }, 加强论证: { ... } }`
   - 合法且不需要引号

3. **图片优化原理**
   - **懒加载**: 只加载视口内的图片
   - **预加载**: 切换前提前加载下一题图片
   - **缓存**: 已加载图片存入内存避免重复请求
   - **重试**: 失败自动重试提高成功率
